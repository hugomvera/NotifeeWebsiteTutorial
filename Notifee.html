Advanced Notifications in React Native with Notifee

React Native Notifee Mobile Dev

Introduction

In order to bring notifications to the next level, we need a powerful tool in React Native. Notifee handles notifications for the most part in both iOS and Android with very similar code.

Setting Up the Project

We need to create the project in CLI and not expo!
hugo@debian: ~/NotifeeArticle bash 
npx @react-native-community/cli@latest init NotifeeDemonstration

run the following command this will install the package for notifee

hugo@debian: ~/NotifeeArticle bash 
hugo@debian:~/NotifeeArticle$npm install --save @notifee/react-native
change directory into the project file
hugo@debian: ~/NotifeeArticle bash 
hugo@debian:~/NotifeeArticle$cd NotifeeDemonstration
now run in terminal metro the react native server
hugo@debian: ~/NotifeeArticle bash 

hugo@debian:~/NotifeeArticle$npx  react-native start

followd by the ios or android emulator in our case we are using Android in a new termnial tab
hugo@debian: ~/NotifeeArticle bash 

hugo@debian:~/NotifeeArticle$ npx react-native run-android

Now we need to go over some of the code. the plan is when we hit the button that will call a funciton that will create the notification . the name of that function is onDisplayNotification(). inside that onDipslayNotifcation function we need to ask for permsision this is mostly for ios // Request permissions (required for iOS) await notifee.requestPermission() and also for android to create a channel // Create a channel (required for Android) const channelId = await notifee.createChannel({ id: 'default', name: 'Default Channel', }); and then we can finaly call the notifcaiton // Display a notification await notifee.displayNotification({ title: 'Notification Title', body: 'Main body content of the notification', android: { channelId, smallIcon: 'name-of-a-small-icon', // optional, defaults to 'ic_launcher'. // pressAction is needed if you want the notification to open the app when pressed pressAction: { id: 'default', }, }, });
